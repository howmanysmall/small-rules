`small-rules/require-react-component-keys` reports code described by its source metadata: `require keys on React components when used in lists or iteration`. It reports with the message `JSX element in list/callback lacks key prop. React Luau warns about missing keys in _G.__DEV__ mode. Add a unique 'key' prop using a stable identifier (not array index).`.

## Rule Details

This rule is configured as `small-rules/require-react-component-keys` and reports the concrete syntax covered by its implementation in `src/rules/require-react-component-keys.ts`.

<Aside type="note">

Rule messages:

```text
missingKey: JSX element in list/callback lacks key prop. React Luau warns about missing keys in _G.__DEV__ mode. Add a unique `key` prop using a stable identifier (not array index).
rootComponentWithKey: Root return has unnecessary key prop. The key gets overwritten by the parent anyway. Remove the `key` prop.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-react-component-keys": [
			"error",
			{
				"allowRootKeys": false,
				"ignoreCallExpressions": ["ReactTree.mount"],
				"iterationMethods": [
					"map",
					"filter",
					"forEach",
					"flatMap",
					"reduce",
					"reduceRight",
					"some",
					"every",
					"find",
					"findIndex"
				],
				"memoizationHooks": ["useCallback", "useMemo"]
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts

function Bad1() {

            <span />

```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
function Good1() {
    return <div />;
}

```

</RuleExample>
