`small-rules/ban-react-fc` reports code described by its source metadata: `ban React.FC and similar component type annotations. Use explicit function declarations instead`. It reports with the message `Avoid React.FC/FunctionComponent/VFC/VoidFunctionComponent types. They break debug information and profiling. Use explicit function declarations instead: 'function Component(props: Props) { ... }'`.

## Rule Details

This rule is configured as `small-rules/ban-react-fc` and reports the concrete syntax covered by its implementation in `src/rules/ban-react-fc.ts`.

<Aside type="note">

Rule message:

```text
banReactFC: Avoid React.FC/FunctionComponent/VFC/VoidFunctionComponent types. They break debug information and profiling. Use explicit function declarations instead: `function Component(props: Props) { ... }`
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/ban-react-fc": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const Component: React.FC = () => {};
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
function Component() {}
```

</RuleExample>
