`small-rules/no-inline-property-on-memo-component` reports code described by its source metadata: `prevent inline properties from being passed to memoized components`. It reports with the message `Inline {{type}} passed to memoized component "{{name}}" — new references cause unnecessary re-renders`.

## Rule Details

This rule is configured as `small-rules/no-inline-property-on-memo-component` and reports the concrete syntax covered by its implementation in `src/rules/no-inline-property-on-memo-component.ts`.

<Aside type="note">

Rule message:

```text
inlineProperty: Inline {{type}} passed to memoized component "{{name}}" — new references cause unnecessary re-renders
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-inline-property-on-memo-component": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts

import { memo } from "@rbxts/react";

const MemoCard = memo(() => <div />);

const view = <MemoCard options={{ enabled: true }} />;

```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts

import { memo } from "@rbxts/react";

function Card() {
	return <div />;
}

const view = <Card options={{ enabled: true }} />;

```

</RuleExample>
