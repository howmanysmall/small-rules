`small-rules/rerender-memo-with-default-value` reports code described by its source metadata: `prevent inline empty object and array defaults in component prop destructuring`. It reports with the message `Default prop value [] creates a new array reference every render — extract to a module-level constant`.

## Rule Details

This rule is configured as `small-rules/rerender-memo-with-default-value` and reports the concrete syntax covered by its implementation in `src/rules/rerender-memo-with-default-value.ts`.

<Aside type="note">

Rule messages:

```text
emptyArrayDefault: Default prop value [] creates a new array reference every render — extract to a module-level constant
emptyObjectDefault: Default prop value {} creates a new object reference every render — extract to a module-level constant
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/rerender-memo-with-default-value": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
function Component({ options = {} }) {
	return options;
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
function component({ options = {} }) {
	return options;
}
```

</RuleExample>
