`small-rules/prefer-context-stack` reports code described by its source metadata: `prefer a local ContextStack component over directly nesting multiple context providers`. It reports with the message `Use the local 'ContextStack' component instead of nesting multiple context providers directly.`.

## Rule Details

This rule is configured as `small-rules/prefer-context-stack` and reports the concrete syntax covered by its implementation in `src/rules/prefer-context-stack.ts`.

<Aside type="note">

Rule message:

```text
preferContextStack: Use the local `ContextStack` component instead of nesting multiple context providers directly.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-context-stack": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import ContextStack from "../providers/context-stack";

export function Example(locale: string, theme: string) {
    return <ThemeContext.Provider value={theme}><LocaleContext.Provider value={locale}><App /></LocaleContext.Provider></ThemeContext.Provider>;
}
```

</RuleExample>

When auto-fixed, this becomes:

```ts
import ContextStack from "../providers/context-stack";

export function Example(locale: string, theme: string) {
    return <ContextStack providers={[<ThemeContext.Provider value={theme} />, <LocaleContext.Provider value={locale} />]}><App /></ContextStack>;
}
```

<RuleExample type="pass" title="Passes">

```ts
import ContextStack from "../providers/context-stack";

export function Example(theme: string) {
    return <ThemeContext.Provider value={theme}><App /></ThemeContext.Provider>;
}
```

</RuleExample>
