`small-rules/prefer-use-reducer` reports code described by its source metadata: `suggest using useReducer for related state updates instead of multiple useState calls`. It reports with the message `Component "{{componentName}}" has {{useStateCount}} useState calls — consider useReducer for related state`.

## Rule Details

This rule is configured as `small-rules/prefer-use-reducer` and reports the concrete syntax covered by its implementation in `src/rules/prefer-use-reducer.ts`.

<Aside type="note">

Rule message:

```text
excessiveUseState: Component "{{componentName}}" has {{useStateCount}} useState calls — consider useReducer for related state
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-use-reducer": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { useState } from "@rbxts/react";

function Component() {
	const [firstValue] = useState(0);
	const [secondValue] = useState(0);
	const [thirdValue] = useState(0);
	const [fourthValue] = useState(0);
	const [fifthValue] = useState(0);
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { useState } from "@rbxts/react";

function Component() {
	const [firstValue] = useState(0);
	const [secondValue] = useState(0);
	const [thirdValue] = useState(0);
	const [fourthValue] = useState(0);
}
```

</RuleExample>
