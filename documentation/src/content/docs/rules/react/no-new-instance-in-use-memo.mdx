`small-rules/no-new-instance-in-use-memo` reports code described by its source metadata: `disallow configured constructor calls (default: new Instance) inside React useMemo callbacks`. It reports with the message `Avoid creating '{{constructorName}}' with 'new' inside useMemo. Create it outside the memo callback or use an effect/ref pattern.`.

## Rule Details

This rule is configured as `small-rules/no-new-instance-in-use-memo` and reports the concrete syntax covered by its implementation in `src/rules/no-new-instance-in-use-memo.ts`.

<Aside type="note">

Rule message:

```text
noNewInUseMemo: Avoid creating '{{constructorName}}' with `new` inside useMemo. Create it outside the memo callback or use an effect/ref pattern.
```

</Aside>

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { useMemo } from "@rbxts/react";

const model = useMemo(() => {
	return new Instance("Model");
}, []);
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
function useMemo(factory) {
	return factory();
}

const model = useMemo(() => new Instance("Model"), []);
```

</RuleExample>
