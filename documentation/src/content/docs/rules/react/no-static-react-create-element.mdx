`small-rules/no-static-react-create-element` reports code described by its source metadata: `require JSX for static React.createElement calls`. It reports with the message `Use JSX instead of static React.createElement calls.`.

## Rule Details

This rule is configured as `small-rules/no-static-react-create-element` and reports the concrete syntax covered by its implementation in `src/rules/no-static-react-create-element.ts`.

<Aside type="note">

Rule message:

```text
useJsx: Use JSX instead of static React.createElement calls.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-static-react-create-element": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import React from "@rbxts/react";

const element = React.createElement("frame");
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import React from "@rbxts/react";

const element = React.createElement(config.component);
```

</RuleExample>
