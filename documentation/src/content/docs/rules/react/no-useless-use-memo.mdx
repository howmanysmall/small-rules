`small-rules/no-useless-use-memo` reports code described by its source metadata: `disallow useMemo calls that only wrap values static enough to live at module scope`. It reports with the message `useMemo is wrapping a static value. Move the value to module scope instead of paying hook overhead for no runtime benefit.`.

## Rule Details

This rule is configured as `small-rules/no-useless-use-memo` and reports the concrete syntax covered by its implementation in `src/rules/no-useless-use-memo.ts`.

<Aside type="note">

Rule message:

```text
uselessUseMemo: useMemo is wrapping a static value. Move the value to module scope instead of paying hook overhead for no runtime benefit.
```

</Aside>

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { useMemo } from "react";
import { AnimationLibrary, SpringConfiguration, getAnimationConfiguration } from "./animation-config";

const rotationConfiguration = useMemo(
	() => getAnimationConfiguration(SpringConfiguration.Sharp, AnimationLibrary.ReactSpring),
	[],
);
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { useMemo } from "react";
import { AnimationLibrary, SpringConfiguration, getAnimationConfiguration } from "./animation-config";

function Component({ theme }) {
	const value = useMemo(() => getAnimationConfiguration(theme, AnimationLibrary.ReactSpring), [theme]);
	return value;
}
```

</RuleExample>
