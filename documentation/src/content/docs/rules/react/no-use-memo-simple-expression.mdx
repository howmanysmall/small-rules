`small-rules/no-use-memo-simple-expression` reports code described by its source metadata: `disallow useMemo for expressions that are already trivial to compute`. It reports with the message `useMemo wrapping a trivially cheap expression - memo overhead exceeds the computation`.

## Rule Details

This rule is configured as `small-rules/no-use-memo-simple-expression` and reports the concrete syntax covered by its implementation in `src/rules/no-use-memo-simple-expression.ts`.

<Aside type="note">

Rule message:

```text
simpleMemo: useMemo wrapping a trivially cheap expression - memo overhead exceeds the computation
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-use-memo-simple-expression": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { useMemo } from "react";

const value = useMemo(() => 1, []);
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { useMemo } from "react";

const value = useMemo(1, []);
```

</RuleExample>
