`small-rules/prefer-padding-components` reports code described by its source metadata: `prefer local EqualPadding and DirectionalPadding components over matching <uipadding /> declarations`. It reports with the message `Use the local 'DirectionalPadding' component when horizontal and vertical padding already match.`.

## Rule Details

This rule is configured as `small-rules/prefer-padding-components` and reports the concrete syntax covered by its implementation in `src/rules/prefer-padding-components.ts`.

<Aside type="note">

Rule messages:

```text
preferDirectionalPadding: Use the local `DirectionalPadding` component when horizontal and vertical padding already match.
preferEqualPadding: Use the local `EqualPadding` component when all four padding values already match.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-padding-components": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { EqualPadding } from "../ui/equal-padding";

export function Example(padding: UDim) {
    return <uipadding PaddingBottom={padding} PaddingLeft={padding} PaddingRight={padding} PaddingTop={padding} />;
}
```

</RuleExample>

When auto-fixed, this becomes:

```ts
import { EqualPadding } from "../ui/equal-padding";

export function Example(padding: UDim) {
    return <EqualPadding padding={padding} />;
}
```

<RuleExample type="pass" title="Passes">

```ts
import { EqualPadding } from "../ui/equal-padding";

export function Example(padding: UDim) {
    return <uipadding PaddingBottom={padding} PaddingLeft={padding} PaddingRight={padding} PaddingTop={padding} />;
}
```

</RuleExample>
