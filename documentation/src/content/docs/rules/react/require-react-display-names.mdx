`small-rules/require-react-display-names` reports code described by its source metadata: `require displayName on exported memo components and contexts`. It reports with the message `Directly exporting createContext() result prevents setting displayName.`.

## Rule Details

This rule is configured as `small-rules/require-react-display-names` and reports the concrete syntax covered by its implementation in `src/rules/require-react-display-names.ts`.

<Aside type="note">

Rule messages:

```text
directContextExport: Directly exporting createContext() result prevents setting displayName.
directMemoExport: Directly exporting memo() result prevents setting displayName.
missingContextDisplayName: Context '{{variableName}}' must have a displayName assigned.
missingMemoDisplayName: Memo component '{{variableName}}' must have a displayName assigned.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-react-display-names": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts

import { memo } from "@rbxts/react";

function ComponentNoMemo() {
    return <div />;
}

export default memo(ComponentNoMemo);

```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts

import { memo } from "@rbxts/react";

function ChecklistClaimButtonNoMemo() {
    return <div />;
}

export const ChecklistClaimButton = memo(ChecklistClaimButtonNoMemo);
ChecklistClaimButton.displayName = "ChecklistClaimButton";
export default ChecklistClaimButton;

```

</RuleExample>
