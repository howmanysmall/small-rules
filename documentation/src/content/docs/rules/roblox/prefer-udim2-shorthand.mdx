`small-rules/prefer-udim2-shorthand` reports code described by its source metadata: `prefer UDim2.fromScale() or UDim2.fromOffset() over new UDim2() when all offsets or all scales are zero`. It reports with the message `Use UDim2.fromOffset() instead of new UDim2() when all scales are 0.`.

## Rule Details

This rule is configured as `small-rules/prefer-udim2-shorthand` and reports the concrete syntax covered by its implementation in `src/rules/prefer-udim2-shorthand.ts`.

<Aside type="note">

Rule messages:

```text
preferFromOffset: Use UDim2.fromOffset() instead of new UDim2() when all scales are 0.
preferFromScale: Use UDim2.fromScale() instead of new UDim2() when all offsets are 0.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-udim2-shorthand": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
new UDim2(1, 0, 1, 0);
```

</RuleExample>

When auto-fixed, this becomes:

```ts
UDim2.fromScale(1, 1);
```

<RuleExample type="pass" title="Passes">

```ts
new UDim2(1, 2, 3, 4);
```

</RuleExample>
