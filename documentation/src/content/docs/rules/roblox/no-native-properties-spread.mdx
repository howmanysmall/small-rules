`small-rules/no-native-properties-spread` reports code described by its source metadata: `disallow spreading static property bags into nativeProperties/*NativeProperties props and onto JSX elements via intermediate objects, because the spread creates a new copied table every render`. It reports with the message `Spreading static property bag {{source}} through an intermediate object creates a new table every render. Inline the full object properties directly on the JSX element.`.

## Rule Details

This rule is configured as `small-rules/no-native-properties-spread` and reports the concrete syntax covered by its implementation in `src/rules/no-native-properties-spread.ts`.

<Aside type="note">

Rule messages:

```text
noElementSpread: Spreading static property bag {{source}} through an intermediate object creates a new table every render. Inline the full object properties directly on the JSX element.
noNativePropertiesSpread: Spreading static property bag {{source}} into {{prop}} creates a new table every render. Pass it directly or inline the full object instead.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-native-properties-spread": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const SOME_CONSTANT = {}; const view = <Frame nativeProperties={{ ...SOME_CONSTANT, Text: "hello" }} />;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
const view = <Frame nativeProperties={SOME_CONSTANT} />;
```

</RuleExample>
