## Rule Details

This rule is configured as `small-rules/no-ianitor-success-access` and reports the concrete syntax covered by its implementation in `src/rules/no-ianitor-success-access.ts`.

## Configuration

<RuleOptions rule="no-ianitor-success-access" />

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { Ianitor } from "@packages/ianitor";
const ids = { a: "a", b: "b" } as const;
const validator = Ianitor.keyOf(ids);
export function isX(value: unknown) {
	return validator(value).success;
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { Ianitor } from "@packages/ianitor";
const ids = { a: "a", b: "b" } as const;
const validator = Ianitor.keyOf(ids);
```

</RuleExample>
