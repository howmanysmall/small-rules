`small-rules/prefer-sequence-overloads` reports code described by its source metadata: `prefer direct ColorSequence and NumberSequence overloads over two-keypoint arrays`. It reports with the message `Use the single-value sequence overload instead of two identical keypoints.`.

## Rule Details

This rule is configured as `small-rules/prefer-sequence-overloads` and reports the concrete syntax covered by its implementation in `src/rules/prefer-sequence-overloads.ts`.

<Aside type="note">

Rule messages:

```text
preferSingleOverload: Use the single-value sequence overload instead of two identical keypoints.
preferTwoPointOverload: Use the two-value sequence overload instead of constructing a two-keypoint array.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-sequence-overloads": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const gradient = new ColorSequence(new Color3(), new Color3());
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const gradient = new ColorSequence(new Color3());
```

<RuleExample type="pass" title="Passes">

```ts
new ColorSequence([
	new ColorSequenceKeypoint(0, Color3.fromRGB(255, 255, 255)),
	new ColorSequenceKeypoint(0.5, Color3.fromRGB(128, 128, 128)),
	new ColorSequenceKeypoint(1, Color3.fromRGB(0, 0, 0)),
]);
```

</RuleExample>
