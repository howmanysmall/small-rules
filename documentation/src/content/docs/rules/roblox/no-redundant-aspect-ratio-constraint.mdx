`small-rules/no-redundant-aspect-ratio-constraint` reports code described by its source metadata: `disallow redundant uiaspectratioconstraint children inside components that already manage their own aspect ratio internally`. It reports with the message `This component already renders a uiaspectratioconstraint internally. Passing one as a child is redundant and will cause layout issues.`.

## Rule Details

This rule is configured as `small-rules/no-redundant-aspect-ratio-constraint` and reports the concrete syntax covered by its implementation in `src/rules/no-redundant-aspect-ratio-constraint.ts`.

<Aside type="note">

Rule message:

```text
redundantAspectRatioConstraint: This component already renders a uiaspectratioconstraint internally. Passing one as a child is redundant and will cause layout issues.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-redundant-aspect-ratio-constraint": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts

const UI_ASPECT_RATIO_CONSTRAINT = <uiaspectratioconstraint AspectRatio={1.5} />;

function LabelSpritesheet({ children }: { children?: React.ReactNode }) {
	return (
		<imagelabel>
			{children}
			<uiaspectratioconstraint AspectRatio={1.5} />
		</imagelabel>
	);
}

const view = (
	<LabelSpritesheet sprite="icon">
		{UI_ASPECT_RATIO_CONSTRAINT}
	</LabelSpritesheet>
);

```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts

function LabelSpritesheet({ children }: { children?: React.ReactNode }) {
	return (
		<imagelabel>
			{children}
			<uiaspectratioconstraint AspectRatio={1.5} />
		</imagelabel>
	);
}

const view = (
	<LabelSpritesheet sprite="icon">
		<uigradient Color={gradient} />
		<textlabel Text="hello" />
	</LabelSpritesheet>
);

```

</RuleExample>
