`small-rules/require-module-level-instantiation` reports code described by its source metadata: `require configured classes to be instantiated at module level only`. It reports with the message `'{{className}}' from '{{importSource}}' must be instantiated at module level only.`.

## Rule Details

This rule is configured as `small-rules/require-module-level-instantiation` and reports the concrete syntax covered by its implementation in `src/rules/require-module-level-instantiation.ts`.

<Aside type="note">

Rule message:

```text
mustBeModuleLevel: '{{className}}' from '{{importSource}}' must be instantiated at module level only.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-module-level-instantiation": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import Log from "@rbxts/rbxts-sleitnick-log";

function useStoryModesState() {
	const log = new Log();
	log.Info("test");
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import Log from "@rbxts/rbxts-sleitnick-log";

const log = new Log();

function useStoryModesState() {
	log.Info("test");
}
```

</RuleExample>
