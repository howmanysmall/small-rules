## Rule Details

This rule is configured as `small-rules/no-array-constructor-elements` and reports the concrete syntax covered by its implementation in `src/rules/no-array-constructor-elements.ts`.

In `roblox-ts` mode, this rule rejects `new Array()` without an explicit generic, element constructor forms like `new Array("a", "b")`, and `new Array<T>(); array.push(...)` patterns that can collapse into an array literal. In `standard` mode, length constructors such as `new Array(5)` are also reported.

## Examples

<RuleExample type="fail" title="Fails">

```ts
const values = new Array("a", "b");
```

```ts
const value = new Array();
```

```ts
const array = new Array<string>();
array.push("a");
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const values = ["a", "b"];
```

<RuleExample type="pass" title="Passes">

```ts
const value = new Set();
```

</RuleExample>
