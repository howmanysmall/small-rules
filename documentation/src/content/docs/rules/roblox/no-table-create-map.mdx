`small-rules/no-table-create-map` reports code described by its source metadata: `disallow map(...) directly on table.create(...) and new Array(...) constructor patterns in roblox-ts`. It reports with the message `Do not map directly on table.create(...) or new Array(...). Allocate first, then write by index in a loop.`.

## Rule Details

This rule is configured as `small-rules/no-table-create-map` and reports the concrete syntax covered by its implementation in `src/rules/no-table-create-map.ts`.

<Aside type="note">

Rule message:

```text
avoidConstructThenMap: Do not map directly on table.create(...) or new Array(...). Allocate first, then write by index in a loop.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-table-create-map": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const mapped = table.create(total).map((_, index) => index);
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
const rewards = new Array<RewardData<RewardType.Item>>(entries);
for (const index of $range(1, entries)) {
	rewards[index - 1] = makeReward(index);
}
```

</RuleExample>
