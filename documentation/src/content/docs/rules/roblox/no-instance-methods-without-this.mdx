`small-rules/no-instance-methods-without-this` reports code described by its source metadata: `detect instance methods that do not use 'this' and suggest converting them to standalone functions for better performance in roblox-ts`. It reports with the message `Method '{{methodName}}' does not use 'this' and creates unnecessary metatable overhead in roblox-ts. Convert it to a standalone function for better performance.`.

## Rule Details

This rule is configured as `small-rules/no-instance-methods-without-this` and reports the concrete syntax covered by its implementation in `src/rules/no-instance-methods-without-this.ts`.

<Aside type="note">

Rule message:

```text
noInstanceMethodWithoutThis: Method '{{methodName}}' does not use 'this' and creates unnecessary metatable overhead in roblox-ts. Convert it to a standalone function for better performance.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-instance-methods-without-this": [
			"error",
			{
				"checkPrivate": true,
				"checkProtected": true,
				"checkPublic": true
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
class MyClass {
	private notifyChanges(value: number): void {
		console.log(value);
	}
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
class MyClass {
	static helper(): void {
		console.log("static");
	}
}
```

</RuleExample>
