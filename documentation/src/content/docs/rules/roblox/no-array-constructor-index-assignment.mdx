## Rule Details

This rule is configured as `small-rules/no-array-constructor-index-assignment` and reports the concrete syntax covered by its implementation in `src/rules/no-array-constructor-index-assignment.ts`.

Detects `const values = new Array<T>()` followed by consecutive index assignments (`values[0] = a; values[1] = b; ...`) and suggests replacing them with an array literal. The rule includes an auto-fix that collapses the pattern into `const values = [a, b, ...]`.

## Examples

<RuleExample type="fail" title="Fails">

```ts
const values = new Array<string>();
values[0] = "a";
values[1] = "b";
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const values = ["a", "b"];
```

<RuleExample type="pass" title="Passes">

```ts
const values = ["a", "b"];
```

```ts
const values = new Array<string>();
// non-contiguous assignment
values[0] = "a";
values[5] = "b";
```

</RuleExample>
