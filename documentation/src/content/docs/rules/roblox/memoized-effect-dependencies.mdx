`small-rules/memoized-effect-dependencies` reports code described by its source metadata: `flags effect dependencies that are not memoized. Unmemoized dependencies can cause unnecessary re-renders or infinite loops`. It reports with the message `{{name}} is not memoized. Wrap it in useMemo/useCallback or move it to module scope.`.

## Rule Details

This rule is configured as `small-rules/memoized-effect-dependencies` and reports the concrete syntax covered by its implementation in `src/rules/memoized-effect-dependencies.ts`.

<Aside type="note">

Rule message:

```text
unmemoizedDependency: {{name}} is not memoized. Wrap it in useMemo/useCallback or move it to module scope.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/memoized-effect-dependencies": [
			"error",
			{
				"environment": "roblox-ts",
				"mode": "definite"
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { useEffect } from "@rbxts/react";

function Component() {
	const dep = {};
	useEffect(() => {}, [dep]);
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { useEffect, useMemo, useCallback } from "@rbxts/react";

function Component() {
	const memo = useMemo(() => ({}), []);
	const callback = useCallback(() => {}, []);
	useEffect(() => {}, [memo, callback]);
}
```

</RuleExample>
