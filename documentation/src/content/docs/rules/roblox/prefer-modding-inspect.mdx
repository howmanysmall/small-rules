`small-rules/prefer-modding-inspect` reports code described by its source metadata: `prefer Modding.inspect over manually enumerating every enum member in a Record<Enum, true>`. It reports with the message `Replace manual {{enumType}} enumeration with Modding.inspect<Record<{{enumType}}, true>>().`.

## Rule Details

This rule is configured as `small-rules/prefer-modding-inspect` and reports the concrete syntax covered by its implementation in `src/rules/prefer-modding-inspect.ts`.

<Aside type="note">

Rule message:

```text
preferModdingInspect: Replace manual {{enumType}} enumeration with Modding.inspect<Record<{{enumType}}, true>>().
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-modding-inspect": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const x: ReadonlyRecord<MyEnum, true> = { a: true, b: true };
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const x = Modding.inspect<Record<MyEnum, true>>();
```

<RuleExample type="pass" title="Passes">

```ts
const x = { a: true, b: true };
```

</RuleExample>
