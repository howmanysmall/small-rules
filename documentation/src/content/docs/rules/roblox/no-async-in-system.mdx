## Rule Details

Planck systems run synchronously. A yielding Roblox API call can pause the system in the middle of a world update, so this rule reports those calls from the active system function.

The rule recognizes functions typed as `PlanckSystem`, `System`, `SystemFunction`, `SystemReturn`, or `SystemTableLike`. It also checks the function returned by a system factory and the `system` property of an object typed with, or satisfying, one of those system types.

Yielding members come from the Roblox API catalog included with the plugin. The rule resolves services imported from `@rbxts/services`, services returned by `game.GetService()`, and Roblox class names supplied by type annotations.

## Configuration

<RuleOptions rule="no-async-in-system" />

Use `additionalSystemTypeNames` if your project wraps Planck with its own system type. `synchronousCallbacks` marks callback arguments that run before the surrounding call returns, which lets the rule follow calls such as a project-specific query helper. `callbackParameterTypes` describes types supplied to callbacks by an imported API when TypeScript syntax does not include the parameter type.

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { Players } from "@rbxts/services";

const socialSystem: SystemFunction = () => {
	Players.GetFriendsAsync(userId);
};
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import { Players } from "@rbxts/services";

const playerCountSystem: SystemFunction = () => {
	const playerCount = Players.GetPlayers().size();
	world.set(resource, playerCount);
};
```

</RuleExample>
