`small-rules/no-warn` reports code described by its source metadata: `use Log instead of warn()`. It reports with the message `warn() is a raw output function lacking log levels, timestamps, and filtering. Production systems require structured logging for debugging and monitoring. Replace warn(...) with Log.`.

## Rule Details

This rule is configured as `small-rules/no-warn` and reports the concrete syntax covered by its implementation in `src/rules/no-warn.ts`.

<Aside type="note">

Rule message:

```text
noWarn: warn() is a raw output function lacking log levels, timestamps, and filtering. Production systems require structured logging for debugging and monitoring. Replace warn(...) with Log.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-warn": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
warn("Hello");
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
Log.warn("Hello");
```

</RuleExample>
