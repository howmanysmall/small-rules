`small-rules/prefer-math-min-max` reports code described by its source metadata: `prefer math.min() and math.max() over simple clamp-like ternaries`. It reports with the message `Use 'math.min()' or 'math.max()' instead of this ternary comparison.`.

## Rule Details

This rule is configured as `small-rules/prefer-math-min-max` and reports the concrete syntax covered by its implementation in `src/rules/prefer-math-min-max.ts`.

<Aside type="note">

Rule message:

```text
preferMathMethod: Use `math.min()` or `math.max()` instead of this ternary comparison.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-math-min-max": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
height > 50 ? 50 : height;
```

</RuleExample>

When auto-fixed, this becomes:

```ts
math.min(height, 50);
```

<RuleExample type="pass" title="Passes">

```ts
math.min(height, 50);
```

</RuleExample>
