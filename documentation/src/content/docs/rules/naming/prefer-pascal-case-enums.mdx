`small-rules/prefer-pascal-case-enums` reports code described by its source metadata: `enforce Pascal case when naming enums`. It reports with the message `Enum '{{ identifier }}' uses non-standard casing. TypeScript convention requires PascalCase for enum names and members to distinguish them from variables (camelCase) and constants (UPPER_CASE). Rename to PascalCase: capitalize first letter of each word, no underscores.`.

## Rule Details

This rule is configured as `small-rules/prefer-pascal-case-enums` and reports the concrete syntax covered by its implementation in `src/rules/prefer-pascal-case-enums.ts`.

<Aside type="note">

Rule message:

```text
notPascalCase: Enum '{{ identifier }}' uses non-standard casing. TypeScript convention requires PascalCase for enum names and members to distinguish them from variables (camelCase) and constants (UPPER_CASE). Rename to PascalCase: capitalize first letter of each word, no underscores.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-pascal-case-enums": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
enum SORTORDER {
	MostRecent,
	LeastRecent,
	Newest,
	Oldest,
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
enum SortOrder {
	MostRecent,
	LeastRecent,
	Newest,
	Oldest,
}
```

</RuleExample>
