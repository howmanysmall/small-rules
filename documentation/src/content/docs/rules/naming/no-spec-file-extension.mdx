`small-rules/no-spec-file-extension` reports code described by its source metadata: `disallow the .spec.{ts,tsx} file extension for test files. Use .test.{ts,tsx} instead`. It reports with the message `Test files must use .test.ts or .test.tsx instead of .spec.ts or .spec.tsx. Rename this file to use the .test extension.`.

## Rule Details

This rule is configured as `small-rules/no-spec-file-extension` and reports the concrete syntax covered by its implementation in `src/rules/no-spec-file-extension.ts`.

<Aside type="note">

Rule message:

```text
noSpecFileExtension: Test files must use .test.ts or .test.tsx instead of .spec.ts or .spec.tsx. Rename this file to use the .test extension.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-spec-file-extension": "error"
	}
}
```
