`small-rules/prevent-abbreviations` reports code described by its source metadata: `prevent abbreviations`. It reports with the message `The {{nameTypeText}} '{{discouragedName}}' should be named '{{replacement}}'. A more descriptive name will do too.`.

## Rule Details

This rule is configured as `small-rules/prevent-abbreviations` and reports the concrete syntax covered by its implementation in `src/rules/prevent-abbreviations.ts`.

<Aside type="note">

Rule messages:

```text
replace: The {{nameTypeText}} `{{discouragedName}}` should be named `{{replacement}}`. A more descriptive name will do too.
suggestion: Please rename the {{nameTypeText}} `{{discouragedName}}`. Suggested names are: {{replacementsText}}. A more descriptive name will do too.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prevent-abbreviations": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const err = new Error();
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const error = new Error();
```

<RuleExample type="pass" title="Passes">

```ts
const ERR = "error";
```

</RuleExample>
