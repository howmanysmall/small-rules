`small-rules/array-type-generic` reports code described by its source metadata: `disallow bracket array type syntax and require Array<T> / ReadonlyArray<T>`. It reports with the message `Bracket array type syntax is not allowed. Use Array<T> or ReadonlyArray<T> generic syntax.`.

## Rule Details

This rule is configured as `small-rules/array-type-generic` and reports the concrete syntax covered by its implementation in `src/rules/array-type-generic.ts`.

<Aside type="note">

Rule message:

```text
useGenericArrayType: Bracket array type syntax is not allowed. Use Array<T> or ReadonlyArray<T> generic syntax.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/array-type-generic": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
type A = string[];
```

</RuleExample>

When auto-fixed, this becomes:

```ts
type A = Array<string>;
```

<RuleExample type="pass" title="Passes">

```ts
type Point = [x: number, y: number];
```

</RuleExample>
