`small-rules/require-async-suffix` reports code described by its source metadata: `require async function names to end with Async`. It reports with the message `Async functions must have names that end with Async.`.

## Rule Details

This rule is configured as `small-rules/require-async-suffix` and reports the concrete syntax covered by its implementation in `src/rules/require-async-suffix.ts`.

<Aside type="note">

Rule message:

```text
missingAsyncSuffix: Async functions must have names that end with Async.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-async-suffix": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
async function getAll(): Promise<string> {
	return "value";
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
async function getAllAsync(): Promise<string> {
	return "value";
}
```

</RuleExample>
