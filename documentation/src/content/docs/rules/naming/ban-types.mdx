`small-rules/ban-types` reports code described by its source metadata: `ban configured TypeScript utility types, defaulting to Omit in favor of Except`. It reports with the message `Type '{{typeName}}' is banned by project configuration. Use the project-preferred alternative for this type.`.

## Rule Details

This rule is configured as `small-rules/ban-types` and reports the concrete syntax covered by its implementation in `src/rules/ban-types.ts`.

<Aside type="note">

Rule messages:

```text
bannedType: Type '{{typeName}}' is banned by project configuration. Use the project-preferred alternative for this type.
bannedTypeWithReplacement: Type '{{typeName}}' is banned. Use '{{replacementName}}' instead.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/ban-types": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
type User = {
	readonly name: string;
	readonly password: string;
};

type HiddenUser = Omit<User, "password">;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
type User = {
	readonly name: string;
};

type ActiveUser = Pick<User, "name">;
```

</RuleExample>
