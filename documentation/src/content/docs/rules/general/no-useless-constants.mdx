`small-rules/no-useless-constants` reports code described by its source metadata: `disallow constants that do not add value`. It reports with the message `Constant '{{name}}' is only referenced once in the same scope. Inline it directly, or move it to a higher scope if reference stability is needed.`.

## Rule Details

This rule is configured as `small-rules/no-useless-constants` and reports the concrete syntax covered by its implementation in `src/rules/no-useless-constants.ts`.

<Aside type="note">

Rule messages:

```text
uselessConstant: Constant '{{name}}' is only referenced once in the same scope. Inline it directly, or move it to a higher scope if reference stability is needed.
uselessConstantNoFix: Constant '{{name}}' is only referenced once in the same scope. It cannot be auto-inlined because the initializer is not safely movable or the declaration has attached comments. Inline it manually, or move it to a higher scope if reference stability is needed.
uselessConstants: Constants '{{names}}' are only referenced once in the same scope. Inline them directly, or move them to a higher scope if reference stability is needed.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-useless-constants": [
			"error",
			{
				"ignoreCallPatterns": ["^Array\\b", "^Object\\b", "^Map\\b", "^Set\\b", "^WeakMap\\b", "^WeakSet\\b"]
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const TITLE_OFFSET = 225;
const TEXT_NATIVE = { Offset: TITLE_OFFSET };
```

</RuleExample>

When auto-fixed, this becomes:

```ts
const TEXT_NATIVE = { Offset: 225 };
```

<RuleExample type="pass" title="Passes">

```ts
const TEXT_NATIVE = { Offset: 225 };
```

</RuleExample>
