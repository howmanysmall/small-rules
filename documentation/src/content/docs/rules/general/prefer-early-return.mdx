`small-rules/prefer-early-return` reports code described by its source metadata: `prefer early returns over full-body conditional wrapping`. It reports with the message `Function body is wrapped in a single conditional without an else branch. This increases nesting depth and cognitive load. Invert the condition and return early: if (!condition) return; then place the main logic at the top level.`.

## Rule Details

This rule is configured as `small-rules/prefer-early-return` and reports the concrete syntax covered by its implementation in `src/rules/prefer-early-return.ts`.

<Aside type="note">

Rule message:

```text
preferEarlyReturn: Function body is wrapped in a single conditional without an else branch. This increases nesting depth and cognitive load. Invert the condition and return early: if (!condition) return; then place the main logic at the top level.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-early-return": [
			"error",
			{
				"maximumStatements": 1
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
function foo() {
	if (something) {
		doSomething();
		doSomethingElse();
	}
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
function foo() {
	if (!something) {
		return;
	}
	doSomething();
	doSomethingElse();
}
```

</RuleExample>
