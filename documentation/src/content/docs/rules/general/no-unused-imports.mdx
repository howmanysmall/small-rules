`small-rules/no-unused-imports` reports code described by its source metadata: `disallow unused imports`. It reports with the message `Import '{{identifierName}}' is defined but never used.`.

## Rule Details

This rule is configured as `small-rules/no-unused-imports` and reports the concrete syntax covered by its implementation in `src/rules/no-unused-imports.ts`.

<Aside type="note">

Rule message:

```text
unusedImport: Import '{{identifierName}}' is defined but never used.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-unused-imports": [
			"error",
			{
				"checkJSDoc": true
			}
		]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import UnusedDefault from "./module";
```

</RuleExample>

When auto-fixed, this becomes:

```ts
import { unused2 } from "./module";
```

<RuleExample type="pass" title="Passes">

```ts
import UsedDefault from "./module";
UsedDefault();
```

</RuleExample>
