`small-rules/directive-disable-enable-pair` reports code described by its source metadata: `require a 'eslint-enable' comment for every 'eslint-disable' comment`. It reports with the message `Requires 'eslint-enable' directive.`.

## Rule Details

This rule is configured as `small-rules/directive-disable-enable-pair` and reports the concrete syntax covered by its implementation in `src/rules/directive-disable-enable-pair.ts`.

<Aside type="note">

Rule messages:

```text
missingPair: Requires 'eslint-enable' directive.
missingRulePair: Requires 'eslint-enable' directive for '{{ruleId}}'.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/directive-disable-enable-pair": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const z = 0;
/* oxlint-disable no-console */
const x = 1;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
// oxlint-disable no-console
const x = 1;
```

</RuleExample>
