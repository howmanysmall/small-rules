`small-rules/directive-no-restricted-disable` reports code described by its source metadata: `disallow 'eslint-disable' comments about specific rules`. It reports with the message `Disabling '{{ruleId}}' is not allowed.`.

## Rule Details

This rule is configured as `small-rules/directive-no-restricted-disable` and reports the concrete syntax covered by its implementation in `src/rules/directive-no-restricted-disable.ts`.

<Aside type="note">

Rule message:

```text
disallow: Disabling '{{ruleId}}' is not allowed.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/directive-no-restricted-disable": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
/* oxlint-disable no-console */
const x = 1;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
/* oxlint-disable no-console */
const x = 1;
/* oxlint-enable no-console */
```

</RuleExample>
