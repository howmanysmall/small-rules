---
title: "No Async Constructor"
description: "Disallow asynchronous operations inside class constructors. Constructors return immediately, so async work causes race conditions, unhandled rejections, and incomplete object states."
---

## Rule Details

This rule is configured as `small-rules/no-async-constructor` and reports the concrete syntax covered by its implementation in `src/rules/no-async-constructor.ts`.

## Configuration

<RuleOptions rule="no-async-constructor" />

## Examples

<RuleExample type="fail" title="Fails">

```ts
class BadAwait {
	constructor() {
		await this.init();
	}
	async init() {}
}
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
class GoodFactory {
	private constructor(private data: string) {}
	static async create(): Promise<GoodFactory> {
		const data = await fetchData();
		return new GoodFactory(data);
	}
}
```

</RuleExample>
