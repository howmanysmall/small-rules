## Rule Details

This rule blocks writes to selected properties on selected objects. It reports direct assignments, compound assignments, and update expressions such as `++` or `--`. Reading the same property is allowed.

Each restriction pairs an object pattern with one or more property patterns. Literal names and glob patterns are supported, so one entry can cover a family of globals or properties. The object must be a direct identifier such as `_G`; the rule does not match a nested expression such as `container._G.__DEV__`.

Computed string properties are checked by default. Set `checkComputed` to `false` if code such as `_G["__DEV__"] = true` should remain outside the rule. Dynamic keys cannot be matched because their property name is unknown during linting.

## Configuration

<RuleOptions rule="no-restricted-property-assignment" />

The `restrictions` array is required. A restriction may include its own diagnostic `message`; otherwise, the rule reports the matched object and property names. `allowFiles` accepts filenames and glob patterns for files that may perform these writes, including basename matches such as `main.server.ts` and paths such as `test/**/*.ts`.

## Examples

<RuleExample type="fail" title="Fails">

```ts
_G.__DEV__ = true;
_G.__DEV__++;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
const developmentMode = _G.__DEV__;
_G.__PROD__ = true;
```

</RuleExample>
