`small-rules/directive-no-use` reports code described by its source metadata: `disallow ESLint directive-comments`. It reports with the message `Unexpected ESLint directive comment.`.

## Rule Details

This rule is configured as `small-rules/directive-no-use` and reports the concrete syntax covered by its implementation in `src/rules/directive-no-use.ts`.

<Aside type="note">

Rule message:

```text
disallow: Unexpected ESLint directive comment.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/directive-no-use": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
/* oxlint-disable no-console */
const x = 1;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
/* oxlint-disable no-console */
const x = 1;
```

</RuleExample>
