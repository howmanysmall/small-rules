`small-rules/require-throw-error-capture` reports code described by its source metadata: `require 'Error.captureStackTrace' before throwing errors to produce accurate stack traces`. It reports with the message `Call 'Error.captureStackTrace' on this error before throwing it so the stack trace points to the throw site.`.

## Rule Details

This rule is configured as `small-rules/require-throw-error-capture` and reports the concrete syntax covered by its implementation in `src/rules/require-throw-error-capture.ts`.

<Aside type="note">

Rule message:

```text
missingCaptureStackTrace: Call 'Error.captureStackTrace' on this error before throwing it so the stack trace points to the throw site.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-throw-error-capture": ["error", {}]
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
function fetchModels() {
	if (!response.ok) throw new Error(`Failed to fetch models: ${response.status} ${response.statusText}`);
}
```

</RuleExample>

When auto-fixed, this becomes:

```ts
function fetchModels() {
	if (!response.ok) {
		const error = new Error(`Failed to fetch models: ${response.status} ${response.statusText}`);
		Error.captureStackTrace(error, fetchModels);
		throw error;
	}
}
```

<RuleExample type="pass" title="Passes">

```ts
throw new Error("top level");
```

</RuleExample>
