`small-rules/prefer-module-scope-constants` reports code described by its source metadata: `prefer that screaming snake case variables always be defined using 'const', and always appear at module scope`. It reports with the message `You must place screaming snake case at module scope. If this is not meant to be a module-scoped variable, use camelcase instead.`.

## Rule Details

This rule is configured as `small-rules/prefer-module-scope-constants` and reports the concrete syntax covered by its implementation in `src/rules/prefer-module-scope-constants.ts`.

<Aside type="note">

Rule messages:

```text
mustBeModuleScope: You must place screaming snake case at module scope. If this is not meant to be a module-scoped variable, use camelcase instead.
mustUseConst: You must use `const` when defining screaming snake case variables. If this is not a constant, use camelcase instead.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/prefer-module-scope-constants": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
let FOO = true;
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
const FOO = true;
```

</RuleExample>
