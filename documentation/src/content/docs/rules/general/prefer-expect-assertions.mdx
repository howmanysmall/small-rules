## Rule Details

This rule is configured as `small-rules/prefer-expect-assertions` and reports the concrete syntax covered by its implementation in `src/rules/prefer-expect-assertions.ts`. It requires assertion guards for tests that contain counted expect/assertion calls, and prefers `expect.assertions(n)` when the number of assertions is statically known.

## Examples

<RuleExample type="fail" title="Fails">

```ts
it("adds numbers", () => {
	expect(add(1, 2)).toBe(3);
});
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
it("adds numbers", () => {
	expect.assertions(1);
	expect(add(1, 2)).toBe(3);
});
```

</RuleExample>
