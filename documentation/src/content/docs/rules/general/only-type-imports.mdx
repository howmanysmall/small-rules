`small-rules/only-type-imports` reports code described by its source metadata: `require all imports to be type-only imports. Benchmarks cannot import non-types because the benchmarker plugin will throw an error`. It reports with the message `Benchmarks must use 'import type' to avoid runtime imports that break the benchmarker plugin.`.

## Rule Details

This rule is configured as `small-rules/only-type-imports` and reports the concrete syntax covered by its implementation in `src/rules/only-type-imports.ts`.

<Aside type="note">

Rule message:

```text
onlyTypeImports: Benchmarks must use `import type` to avoid runtime imports that break the benchmarker plugin.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/only-type-imports": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
import { Foo } from "./bar";
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
import type { Foo } from "./bar";
```

</RuleExample>
