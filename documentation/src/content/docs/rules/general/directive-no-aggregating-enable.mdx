`small-rules/directive-no-aggregating-enable` reports code described by its source metadata: `disallow 'eslint-enable' comments for multiple 'eslint-disable' directives`. It reports with the message `'{{kind}}' comment enables rules for multiple disable directives. Please move '{{kind}}' after each disable directive.`.

## Rule Details

This rule is configured as `small-rules/directive-no-aggregating-enable` and reports the concrete syntax covered by its implementation in `src/rules/directive-no-aggregating-enable.ts`.

<Aside type="note">

Rule message:

```text
aggregatingEnable: '{{kind}}' comment enables rules for multiple disable directives. Please move '{{kind}}' after each disable directive.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/directive-no-aggregating-enable": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
/* oxlint-disable no-console */
const x = 1;
/* oxlint-disable no-alert */
const y = 2;
/* oxlint-enable no-console, no-alert */
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
/* oxlint-disable no-console */
const x = 1;
/* oxlint-enable no-console */
```

</RuleExample>
