`small-rules/no-commented-code` reports code described by its source metadata: `disallow commented-out code`. It reports with the message `Commented-out code creates confusion about intent and clutters the codebase. Version control preserves history, making dead code comments unnecessary. Delete the commented code entirely. If needed later, retrieve it from git history.`.

## Rule Details

This rule is configured as `small-rules/no-commented-code` and reports the concrete syntax covered by its implementation in `src/rules/no-commented-code.ts`.

<Aside type="note">

Rule message:

```text
commentedCode: Commented-out code creates confusion about intent and clutters the codebase. Version control preserves history, making dead code comments unnecessary. Delete the commented code entirely. If needed later, retrieve it from git history.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/no-commented-code": [
			"error",
			{
				"maxLines": 0
			}
		]
	}
}
```
