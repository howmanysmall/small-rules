`small-rules/require-unicode-regex` reports code described by its source metadata: `require the 'u' or 'v' unicode flag on arktype regex() calls`. It reports with the message `Missing the 'u' or 'v' unicode flag on this regex() call. Use the unicode flag to avoid silently creating invalid regex patterns.`.

## Rule Details

This rule is configured as `small-rules/require-unicode-regex` and reports the concrete syntax covered by its implementation in `src/rules/require-unicode-regex.ts`.

<Aside type="note">

Rule message:

```text
requireUnicodeFlag: Missing the 'u' or 'v' unicode flag on this regex() call. Use the unicode flag to avoid silently creating invalid regex patterns.
```

</Aside>

## Configuration

```json
{
	"plugins": ["@pobammer-ts/small-rules"],
	"rules": {
		"small-rules/require-unicode-regex": "error"
	}
}
```

## Examples

<RuleExample type="fail" title="Fails">

```ts
const x = regex("foo");
```

</RuleExample>

<RuleExample type="pass" title="Passes">

```ts
const x = regex("foo", "u");
```

</RuleExample>
